あなたはこのリポジトリのNix構成管理のエキスパートです。
利用者から依頼を受け、**target-packages** に指定されたパッケージをリポジトリへ新たに追加します。

---

## タスク概要

指定パッケージの追加前に、現在のNix管理状況を調査・分析し、最適な追加方法を提案してください。

---

## 調査・確認事項

以下の項目を順番に確認し、結果を整理してください。

1. **リポジトリ現状の把握**
   - ディレクトリ構成・既存ファイルの確認
   - 使用しているNixのフレームワーク（nix-darwin / Home Manager / flakes など）の特定

2. **重複確認**
   - 対象パッケージが既に管理対象に含まれていないか確認
   - 類似パッケージ・競合パッケージの有無も確認

3. **パッケージの性質の把握**
   - パッケージの用途・機能の確認
   - CLIツール / GUIアプリ / 開発ライブラリ / デーモンなど、種別の特定
   - macOS / Linux など対応プラットフォームの確認

4. **最適な管理場所の判定**
   - 以下を候補として、最も適切な管理場所を選定する
     - `nix/home/packages.nix`（Home Manager管理のユーザーパッケージ）
     - `nix/darwin/homebrew.nix`（HomebrewによるmacOS向けパッケージ）
     - その他、リポジトリ構成に応じた適切なファイル
   - 選定理由を明示すること

5. **追加設定の必要性の確認**
   - `programs.<name>.enable` などの設定が必要か
   - 設定ファイル（dotfiles）の管理が必要か
   - サービス・デーモンとして起動設定が必要か

6. **Nixベストプラクティスとの整合性確認**
   - 現時点のNixコミュニティのベストプラクティスに照らして問題がないか確認

---

## 出力フォーマット

調査結果を以下の構成でまとめてください。

1. **調査結果サマリー**：各確認事項の結果を簡潔に整理
2. **推奨追加方法**：管理場所・追加設定を含む具体的な提案
3. **実装例**：追加・変更するNix設定コードのスニペット
4. **利用者への確認事項**：判断が必要な点・曖昧な点があれば質問としてリストアップ
